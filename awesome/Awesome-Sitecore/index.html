    Awesome Sitecore

    Awesome Sitecore ¶

    > GitHub上与Sitecore相关的项目的精选列表. 请贡献！

    Sitecore是一个数字平台，可帮助从一个地方管理多个网站. 它提供了营销工具，可以从不同的设备（CRM，跟踪，POS等）传递有关客户的信息. Sitecore允许通过不同的渠道（台式机，手机，API，社交媒体等）访问创建的内容. 该平台在设计上非常灵活，具有许多扩展点，可以在内部部署和云部署. 下面的列表为可从中受益的Sitecore开发人员分类了可扩展性和部署选项.

    欢迎捐款. 通过拉取请求添加链接或创建问题以开始讨论.

    Analytics¶

    • Sitecore Goal Description -在体验资料中显示目标说明.
    • Sitecore Goal Conversion Demo -演示如何获取访问者互动过程中触发的目标，当他们的会话结束后，可以将这些数据发送到外部系统，以便通过向市场营销人员告知客户和潜在客户在其网站上的活动来帮助营销人员.
    • Custom Timeline Eras -在Sitecore的“体验档案”时间轴上演示如何触发结果并显示自定义结果.

    Audit and performance¶

    • Skillcore.Stats -收集所有Sitecore MVC渲染和API调用的详细时间信息.
    • Sitecore.Boost -该存储库包含大量潜在的补丁程序，可用于改善Sitecore平台的性能，以提高性能，其中大部分在Sitecore的生产实例中运行.
    • Sitecore.Cleanup -该模块包含3个代理，它们将分别监视“事件队列”，“发布队列”和“历史记录”表，以确保它们不超过设置的阈值.
    • SitecoreDXG: The Documentation Experience Generator -SitecoreUML系列的Sitecore模板架构可视化和文档编制工具. 利用与StarUML相关联的开源API（SitecoreUML也基于该API），SitecoreDXG提供了从任何具有SitecoreUML的现有Sitecore实例（均支持PaaS和非PaaS）生成模板数据模型架构图和HTML文档的支持. Sitecore服务.
    • sitecore-assembly-lists -Sitecore从8.2 Update 5开始，开始提供Sitecore版本的程序集列表信息.该项目提供了用于在安装过程和开发过程中验证Sitecore程序集列表的工具.

    Azure¶

    • ARM templates for Sitecore -Sitecore的所有当前可用的Azure资源管理器模板.
    • Sitecore Azure Content -有关如何利用Azure服务自动将Sitecore解决方案部署到Microsoft Azure云平台的文章.
    • Sitecore Advanced Azure Devops -使用灵活和通用的脚本在Azure上扩展Sitecore（需要有效的Powershell Azure会话才能连接并可用）.
    • Language Understanding (LUIS) Samples -语言理解智能服务（LUIS）的示例.
    • Sitecore Media Azure Blob Storage provider -允许将Sitecore媒体库资产存储在Azure Blob存储帐户中.
    • CloudMediaLibrary -允许Sitecore媒体库从Cloud Storage存储和提供服务.
    • SitecoreCognitiveServices -将Microsoft的认知服务集成到Sitecore中的SDK.
    • Sitecore Publishing Service Azure Templates -Azure资源管理器（ARM）模板，用于在Sitecore Azure PaaS环境中安装Sitecore发布服务.
    • SitecoreDiagnostics -使用Application Insights进行Sitecore诊断.
    • sitecore-application-insights-annotations -创建App Insinghts注释.

    Cache¶

    • CacheTuner -该模块是Sitecore缓存指南中给出的规则的实现.
    • Cache-Processing-Instance -专用HTML缓存处理实例的概念验证.
    • ExperienceEditorCache -缓存模块，使使用Sitecore Experience Editor的工作更加愉快.
    • ManualHtmlCacheClearer -从Sitecore客户端清除本地和远程实例上的特定HTML缓存.
    • CacheCounters -将Sitecore缓存大小信息报告给Windows性能计数器.
    • Sitecore Rules-Based Output Caching -提供基于规则的Sitecore输出缓存差异支持，从而实现个性化组件和其他需要更复杂的缓存差异逻辑的组件的缓存.

    Commerce¶

    • Reference Storefront -Sitecore参考店面.
    • Commerce Sample Plugins -Sitecore Commerce插件的示例代码（针对XC 9.0.3和9.1）.
    • HabitatHome Commerce -遵循Helix体系结构原理在XP和XC上使用SXA构建的示例.
    • sitecore-commerce-configuration -用于配置和增强Sitecore Commerce Engine配置的工具.
    • Plumber for Sitecore Commerce -查看Sitecore Commerce Engine配置的工具.
    • Short Confirmation Codes -提供六个字符确认代码（可配置），可随机生成并在商务数据库中进行跟踪以避免冲突，因为较短的代码更易于通过支持电话进行交流，并且对于某些客户而言，比提供的25个字符确认更为方便的盒子.

    • Search Index Builder -用于从Sitecore Web应用程序外部重建搜索索引的工具，特别适合于运行时间较长的构建.
    • SolrCloud Install Scripts -PowerShell脚本库可帮助在Windows上安装SolrCloud群集.
    • Common Sitecore ContentSearch extensions -Sitecore contentSearch的常见扩展； 下面的PopulateSolrSchema管道处理器，以确保将Sitecore中所有已安装的语言添加到solr托管模式中.

    • sitecore-azure-search-compat -Sitecore + Azure搜索的兼容性补丁.

    Deployment¶

    • PostDeploySteps - Post deploy steps for use with TDS. 1) TDS Example project that triggers the post deploy step; 2) Sample web project to show pulling in the deploy step; 3) DeployStep class library that you can add to your solution.

    • Sitecore Devops with AppVeyor -示例解决方案，显示了如何开发开源Sitecore模块并将其与AppVeyor挂钩，AppVeyor是托管在云中的持续交付解决方案.

    • Copy Version module -将命令添加到“内容编辑器”，该命令允许用户复制和粘贴项目的最新版本，而不是整个项目. 安装后，可从内容树的上下文菜单中访问“复制版本”和“粘贴版本”命令.

    • Browse Command -添加了一个命令，该命令允许用户在新的浏览器窗口中打开Sitecore内容树中的项目. 除了可以进入预览模式的“预览”命令外，Sitecore不提供此功能，因此给定的模块将执行完全相同的操作，但不进入“预览”模式.
    • Expand Descendants Command -该模块添加了一个命令，该命令允许用户扩展Sitecore内容树中某个项目的所有后代.
    • Environment Styler for Sitecore -登录屏幕和Sitecore标头功能区上特定于环境的样式（和文本）.
    • InsertOptionsLoophole -防止Sitecore用户绕过插入选项.
    • ScopeToThis -允许在Sitecore内容编辑器树上使用Visual Studio的“此功能”.
    • DeviceEditorShortcuts -提高了设备​​编辑器的生产率，显示了如果将数据源设置为该特定组件，并从弹出的单击中预览该数据源.
