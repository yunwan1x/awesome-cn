    NLP with Ruby

    [RubyML | RubyDataScience | RubyInterop]

    >在Ruby中进行文本处理的有用资源

    该精选清单包括 awesome 有关文本计算处理的资源，库，信息源 在人类语言中 Ruby programming language. 该字段通常称为 NLP, Computational Linguistics, HLT （人类语言技术） 并可以与 Artificial Intelligence, Machine Learning, Information Retrieval, Text Mining, Knowledge Extraction 和其他相关学科.

    该列表来自我们在语言模型和NLP工具方面的日常工作. Read why 这个列表很棒. 我们 FAQ 描述 您可能会对重要的决定和有用的答案感兴趣.

    ：sparkles：每个 contribution 不客气！ 通过拉添加链接 请求或提出问题以开始讨论.

    跟着我们 Twitter and please spread the word using the #RubyNLP hash tag!

    NLP Pipeline Subtasks¶

    NLP管道以纯文本开头.

    Pipeline Generation¶

    • composable_operations - 操作管道的定义框架.
    • ruby-spark - 使用易于理解的DSL进行Spark绑定.
    • phobos - 简化的Ruby客户端 Apache Kafka.
    • parallel - 在多个CPU或多个线程上并行执行的主管.
    • pwrake - Rake扩展程序可并行运行本地和远程任务.

    Multipurpose Engines¶

    • open-nlp - Ruby绑定 OpenNLP 工具包.
    • stanford-core-nlp - 斯坦福大学的Ruby绑定 CoreNLP 工具.
    • treat - Ruby的自然语言处理框架（例如 NLTK 对于Python）.
    • nlp_toolz - 包装一些 OpenNLP 类和 原本 Berkeley Parser.
    • open_nlp - JRuby绑定 OpenNLP 工具包.

    On-line APIs¶

    • alchemyapi_ruby - 适用于AlchemyAPI / Bluemix的旧版Ruby SDK.
    • wit-ruby - Ruby客户端库 Wit.ai 语言理解平台.
    • wlapi -的Ruby客户端库 Wortschatz Leipzig 网页服务.
    • monkeylearn-ruby -情绪 分析，主题建模，语言检测，通过命名实体识别 一个基于Ruby的Web API客户端.
    • google-cloud-language - Google的Ruby自然语言服务API.

    Language Identification¶

    语言识别是每个NLP管道中的首要关键步骤之一.

    • scylla - 语言分类和识别.

    Segmentation¶

    用于标记化，单词和句子边界检测和消除歧义的工具.

    • tokenizer - 简单的多语言标记器. [tutorial]
    • pragmatic_tokenizer - 多语言令牌生成器，用于将字符串拆分为令牌.
    • nlp-pure - 在纯Ruby中以最小的依赖关系实现的自然语言处理算法.
    • textoken - 简单和可自定义的文本标记化库.
    • pragmatic_segmenter - 单词边界消除歧义与许多cookie.
    • punkt-segmenter - Punkt Segmenter的纯Ruby实现.
    • tactful_tokenizer - 基于RegExp的令牌生成器，适用于不同的语言.
    • scapel - 句子边界消歧工具.

    Lexical Processing¶

    Stemming¶

    Stemming is the term used in information retrieval to describe the process for 将字形简化为基本表示形式. 区分茎 from Lemmatization 因为`stems'不一定有 语言动机.

    • ruby-stemmer - Ruby-Stemmer向Ruby公开了SnowBall API.
    • uea-stemmer - 用于搜索和索引的保守词干.

    Lemmatization¶

    词法化被认为是寻找单词基本形式的过程. 引理 通常收集在字典中.

    • lemmatizer - 基于WordNet的英语文本Lemmatizer.

    Lexical Statistics: Counting Types and Tokens¶

    • wc - 用于统计文本中单词出现次数的功能.
    • word_count - “字符串”和“哈希”对象的字计数器.
    • words_counted - 纯Ruby库使用不同的自定义选项来统计单词统计信息.

    Filtering Stop Words¶

    • stopwords-filter -过滤和 基于SnowBall lemmatizer停止Word词典.

    Phrasal Level Processing¶

    • n_gram - N-Gram发生器.
    • ruby-ngram - 将单词和短语分解为ngram.
    • raingrams - 用纯Ruby编写的灵活通用的ngrams库.

    Syntactic Processing¶

    Constituency Parsing¶

    • stanfordparser - Stanford Parser的基于Ruby的包装器.
    • rley - 纯Ruby的实现 Earley 无上下文选区语法的解析算法.
    • rsyntaxtree - 基于Ruby的语法树的可视化 RMagick. [dep: ImageMagick]

    Semantic Analysis¶

    • amatch - 字符串之间的五种距离类型集（包括Levenshtein，Sellers，Jaro-Winkler，“对距离”）.
    • damerau-levenshtein - 使用Damerau-Levenshtein算法计算编辑距离.
    • hotwater - 快速的Ruby FFI字符串编辑距离算法.
    • levenshtein-ffi - 使用Damerau-Levenshtein算法快速进行字符串编辑距离计算.
    • tf_idf - 纯Ruby中的术语“频率/文档反向频率”.
    • tf-idf-similarity - 使用TF / IDF计算文本之间的相似度.

    Pragmatical Analysis¶

    • SentimentLib - 简单可扩展的情感分析工具.

    High Level Tasks¶

    Spelling and Error Correction¶

    • gingerice - 通过进行拼写和语法更正 Ginger 火.
    • hunspell-i18n - Ruby绑定到标准 Hunspell 拼写检查器.
    • ffi-hunspell - 基于FFI的Ruby绑定 Hunspell.
    • hunspell - Ruby绑定到 Hunspell 通过Ruby C API.

    Text Alignment¶

    • alignment - 双语文本的对齐例程（Gale-Church实现）.

    Machine Translation¶

    • google-api-client - Google API Ruby客户端.
    • microsoft_translator - 微软翻译器API的Ruby客户端.
    • termit - 在您的终端中使用语音合成功能进行Google翻译.
    • zipf - BLEU和其他基本算法的实现.

    Sentiment Analysis¶

    • stimmung - 语义极性基于 SentiWS 词典.

    Numbers, Dates, and Time Parsing¶

    • chronic - 纯Ruby自然语言日期解析器.
    • chronic_between - 用于日期和时间范围的简单Ruby自然语言解析器.
    • chronic_duration - 纯Ruby解析器用于经过的时间.
    • kronic - 解析和格式化人类可读日期的方法.
    • nickel - 从自然措辞的文本中提取日期，时间和消息信息.
    • tickle - 解析器，用于重复发生和重复的事件.
    • numerizer - Ruby解析器，用于英文数字表达式.

    Named Entity Recognition¶

    • ruby-ner - 由Stanford NER和Ruby命名为Entity Recognition.
    • ruby-nlp - Stanford Pos-Tagger和名称实体识别器的Ruby绑定.

    Text-to-Speech-to-Text¶

    • espeak-ruby - 小型Ruby API，用于利用“ espeak”和“ lame”创建文本到语音的mp3文件.
    • tts - 使用Google翻译服务进行文本到语音的转换.
    • att_speech - AT＆T语音API上的Ruby包装器，用于语音到文本.
    • pocketsphinx-ruby - Pocketsphinx bindings.

    Dialog Agents, Assistants, and Chatbots¶

    • chatterbot - 基于红宝石的直接Twitter Bot Framework，使用OAuth进行身份验证.
    • lita - 高度可扩展的聊天操作机器人框架，使用持久存储编写 Redis.

    Linguistic Resources¶

    • rwordnet - 适用于的纯Ruby自包含API库 Princeton WordNet®.
    • wordnet - 性能调整的绑定 Princeton WordNet®.

    Machine Learning Libraries¶

    Machine Learning 演算法 用纯Ruby或其他编程语言编写，并带有适当的绑定 露比

    有关最新列表，请查看[带Ruby的Awesome ML] [ml-with-ruby]列表.

    • rb-libsvm - 用Ruby支持矢量机.
    • weka - Weka的JRuby绑定，通过Weka实现的不同ML算法.
    • decisiontree - 纯Ruby中的决策树ID3算法 [post].
    • rtimbl - Timbl框架中基于记忆的学习者.
    • classifier-reborn - 通用分类器模块，允许贝叶斯和其他类型的分类.
    • lda-ruby - Ruby的实现 LDA （潜在Dirichlet分配），用于自动主题建模和文档聚类.
    • liblinear-ruby-swig - Ruby与LIBLINEAR的接口（在文本分类方面比LIBSVM效率更高）.
    • linnaeus - Redis支持的贝叶斯分类器.
    • maxent_string_classifier - 基于OpenNLP Maxent框架的用于字符串数据的JRuby最大熵分类器.
    • naive_bayes - 简单的朴素贝叶斯分类器.
    • nbayes - Naive Bayes的全功能Ruby实现.
    • omnicat - 用于文本分类的通用机架框架.
    • omnicat-bayes - 朴素贝叶斯文本分类实施作为OmniCat分类器策略.
    • ruby-fann - Ruby绑定到 Fast Artificial Neural Network Library (FANN).
    • rblearn -特征提取和交叉验证库.

    Data Visualization¶

    请参考 Data Visualization [使用Ruby的数据科学] [ds-with-ruby]列表中的“参考”部分.

    Optical Character Recognition¶

    • tesseract-ocr - 基于FFI的包装器 Tesseract OCR Engine.

    Text Extraction¶

    • yomu - 用于从文件和文档中提取文本和元数据的库 使用 Apache Tika 内容分析工具包.

    Full Text Search, Information Retrieval, Indexing¶

    • rsolr - Ruby和Rails客户端库 Apache Solr.
    • sunspot - 以Rails为中心的客户端 Apache Solr.
    • thinking-sphinx - Active Record 使用的插件 Sphinx 在（不仅是）基于Rails的项目中.
    • elasticsearch - Ruby客户端和API Elasticsearch.
    • elasticsearch-rails - 的Ruby和Rails集成 Elasticsearch.
    • google-api-client - 的Ruby API库 Google 服务.

    Language Aware String Manipulation¶

    库用于语言感知的字符串操作，例如搜索，模式匹配， 大小写转换，代码转换，需要以下信息的正则表达式 基本语言.

    • fuzzy_match - 具有距离测度和正则表达式的模糊字符串比较.
    • fuzzy-string-match - Ruby的模糊字符串匹配库.
    • active_support - RoR ActiveSupport gem具有可以处理大小写的各种字符串扩展名.
    • fuzzy_tools - Ruby中用于模糊搜索的工具集已针对准确性进行了调整.
    • u - U扩展了Ruby的Unicode支持.
    • unicode - Unicode规范化库.
    • CommonRegexRuby - 在字符串中查找许多常见信息.
    • regexp-examples - 生成与给定正则表达式匹配的字符串.
    • verbal_expressions - 使困难的正则表达式变得容易.
    • translit_kit - 将希伯来语和意第绪语文本音译成拉丁字符.
    • re2 - 用于文本挖掘和文本提取的高速正则表达式库.
