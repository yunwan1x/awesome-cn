    Tools¶

    Analyzers¶

    • Web.dev - Get the web's modern capabilities on your own sites and apps with useful guidance and analysis from web.dev.
    • Confess - 使用 PhantomJS 无头分析网页并生成清单.
    • DebugBear - DebugBear 是一个基于 Lighthouse 的站点监控工具. 了解您的分数和指标如何随时间变化，重点了解导致每次变化的原因. DebugBear 是一款付费产品，可免费试用 30 天.
    • Page Speed - PageSpeed 系列工具旨在帮助您优化网站的性能. PageSpeed Insights 产品将帮助您确定可应用于您的网站的性能最佳实践，而 PageSpeed 优化工具可以帮助您自动化该过程.
    • Lighthouse - 渐进式 Web 应用程序的审计和性能指标工具.
    • Screpy - 基于人工智能的 SEO 分析和监控工具
    • YSlow - YSlow analyzes web pages and suggests ways to improve their performance based on a set of rules for high performance web pages.
    • YSlow for PhantomJS - PhantomJS 的 YSlow 还为自动化测试框架引入了新的输出格式：TAP（测试任何协议）和 JUnit.
    • Grunt-WebPageTest - 用于连续测量 WebPageTest 的 Grunt 插件. (Demo)
    • Grunt-yslow - Grunt task for testing page performance using PhantomJS, a headless WebKit browser.
    • Grunt-perfbudget - 用于执行性能预算的 Grunt.js 任务（more on performance budgets).
    • Web Tracing Framework - Web Tracing Framework 是一组库、工具和可视化工具，用于跟踪和调查复杂的 Web 应用程序
    • Yandex.Tank - An extensible open source load testing tool for advanced linux users which is especially good as a part of an automated load testing suite.
    • Yellow Lab Tools - 审核前端不良做法、揭示性能问题和分析 JavaScript 的在线快速简便工具.
    • Pagelocity - A web performance optimization and analysis tool.
    • Varvy - 测试您的网站，看看它是否遵循 Google 的速度指南.
    • Web Bloat Score Calculator - 将页面大小与同一页面的压缩图像进行比较
    • Speed Racer - 使用无头 Chrome 为您的库/应用程序收集性能指标.
    • Speedrank - Speedrank 在后台监控您网站的性能. 它随着时间的推移显示 Lighthouse 报告并提供改进建议. Speedrank 是一款有 14 天试用期的付费产品.

    Analyzers - API¶

    • Node-yslowjs - YSlow.js on Node.js is a simple Node.js wrapper for programmatically running phantomjs yslow.js.
    • PSI - PageSpeed Insights for Node.js - with reporting.

    Application Performance Monitoring¶

    • Datadog APM - 与所有遥测相关的大规模端到端分布式跟踪和 APM.
    • BetterUptime - 一个不错的网站监控工具（捆绑状态页面、事件通知）
    • Nodown - 也是一个监控工具，但侧重于功能测试
    • Pingdom - 获取网站正常运行时间的工具（使用来自不同位置的探测器）
    • UptimeRobot - 另一个正常运行时间监控工具（带有慷慨的免费计划）

    Bundle Analyzer¶

    • Bundlesize - 检查您的捆绑包大小.
    • source-map-explorer - 通过源映射分析和调试捆绑空间使用情况.
    • Bundlephobia - 帮助您找到将 npm 包添加到前端包的性能影响.
    • Webpack bundle analyzer - webpack 插件和 CLI 实用程序，将捆绑内容表示为方便的交互式可缩放树状图.
    • Disc - 可视化browserify项目包的模块树并追踪膨胀.
    • Lasso-analyzer - 分析和可视化 Lasso 创建的项目包.
    • Packtracker.io - 检查每个拉取请求的 webpack 包统计信息，并随着时间的推移跟踪它们.
    • Compression webpack plugin - Prepare compressed versions of assets to serve them with Content-Encoding.
    • BundleStats - 生成捆绑报告（捆绑大小、资产、模块、包）并比较不同构建之间的结果

    Benchmark - CSS¶

    • CSS-perf - 测试 CSS 性能的完全不科学的方法. 大多数这些测试将围绕确定有效 CSS 架构的方法和技术展开. 换句话说，我想知道给定 CSS 策略的特定比较什么最有效.

    Benchmark - JavaScript¶

    > 一组用于创建测试用例和比较 JavaScript 中不同实现的工具.

    • JSPerf - jsPerf 旨在提供一种创建和共享测试用例的简单方法，通过运行基准比较不同 JavaScript 片段的性能. 暂时退出，关注本期更新：https://github.com/jsperf/jsperf.com/issues/537
    • Benchmark.js - A robust benchmarking library that works on nearly all JavaScript platforms, supports high-resolution timers, and returns statistically significant results.
    • JSlitmus - JSLitmus 是一个轻量级的工具，用于创建临时 JavaScript 基准测试.
    • Matcha - Matcha allow you to design experiments that will measure the performance of your code. It is recommended that each bench focus on a specific point of impact in your application.
    • Timing.js - Timing.js 是一小组帮助程序，用于使用 Navigation Timing API 来确定您的应用程序在哪里花费时间. 可用作独立脚本、DevTools Snippet 或书签.
    • Stats.js - 此类提供了一个简单的信息框，可帮助您监控代码性能.
    • PerfTests - JavaScript 继承模型的性能测试.
    • Memory-stats.js - 通过性能内存对 JS 堆大小进行最小监控.

    Benchmark - PHP¶

    • PHPench - PHPench 为 PHP 基准测试创建图形输出. 使用 GnuPlot 实时绘制任何函数的运行时间，并根据结果创建图像.
    • php-bench - 基准测试和分析 PHP 代码块，同时测量性能足迹.

    Bookmarklets¶

    • Yahoo YSlow for Mobile/Bookmarklet - YSlow analyzes web pages and suggests ways to improve their performance based on a set of rules for high performance web pages.
    • PerfMap - 一个小书签，用于使用 Resource Timing API 创建浏览器中加载的资源的前端性能热图.
    • DOM Monster - 一个跨平台、跨浏览器的书签，它将分析您所在页面的 DOM 和其他功能，并为您提供其健康状况.
    • CSS Stress - CSS Stress 是一种测试和性能分析.
    • Performance-Bookmarklet - 通过 Resource Timing API、Navigation Timing API 和 User-Timing 分析当前页面 - 一种轻量级的实时 WebPageTest. 作为 Chrome Extension and Firefox Add-on 以 Performance-Analyser 为名.

    CDN¶

    > 内容分发网络或内容分发网络 (CDN) 是部署在 Internet 上多个数据中心的大型分布式服务器系统. CDN 的目标是以高可用性和高性能为最终用户提供内容. 查看 CDN 的大列表 Wikipedia.

    • Cloudflare CDN - A content delivery network that uses next-gen tech to deliver a fast, reliable, CDN services
    • PageCDN - A state of the art opensource CDN with aggressive content optimization using brotli-11 compression, HTTP/2 server push, better HTTP/2 multiplexing, and more. Supports 100s of libraries and 2000+ WordPress themes already. Easy to use, easy to link and very fast.
    • jsDelivr - 与 Google 托管库类似，jsDelivr 是一个开源 CDN，允许开发人员托管他们自己的项目，任何人都可以链接到我们在其网站上托管的文件.
    • Google Hosted Libraries - Google Hosted Libraries 是最流行的开源 JavaScript 库的内容分发网络.
    • CDNjs - An open source CDN for JavaScript and CSS sponsored by CloudFlare that hosts everything from jQuery and Modernizr to Bootstrap.
    • Amazon CloudFront - 亚马逊的内容交付网络，可以与其他亚马逊服务很好地集成，也可以单独使用.
    • jQuery - jQuery CDN – 最新稳定版本，由 MaxCDN 提供支持.
    • UpYun CDN - 由 upyun 提供的 CDN.
    • Bootstrap 中文网开放 CDN 服务 - Bootstrap中文网开放CDN服务（仅HTTP）.
    • Yandex CDN - Yandex 内容交付网络托管流行的第三方 JavaScript 和 CSS 库（最适合在俄罗斯使用）.

    Specs¶

    • Web Performance Working Group - The mission of the Web Performance Working Group, part of the Rich Web Client Activity, is to provide methods to measure aspects of application performance of user agent features and APIs.
    • Page Visibility - 该规范定义了一种方法，供站点开发人员以编程方式确定页面的当前可见性状态，以便开发功率和 CPU 高效的 Web 应用程序.
    • Navigation Timing - 该规范定义了一个统一的接口来存储和检索与文档导航相关的高分辨率性能度量数据.
    • Resource Timing - 该规范定义了一个接口，供 Web 应用程序访问文档中资源的完整时间信息.
    • User Timing - 该规范定义了一个接口，帮助 Web 开发人员通过访问高精度时间戳来衡量其应用程序的性能.
    • Performance Timeline - This specification defines a unified interface to store and retrieve performance metric data. This specification does not cover individual performance metric interfaces.
    • CSS will-change - 该规范定义了 will-change CSS 属性，它允许作者提前声明哪些属性可能在未来发生变化，因此 UA 可以在需要它们之前的某个时间设置适当的优化. 这样，当实际更改发生时，页面会以快速的方式更新.
    • Resource Hints - 本规范定义了 HTML 链接元素 (<link>) 的 dns-prefetch、preconnect、prefetch 和 prerender 关系. 这些原语使开发人员和生成或交付资源的服务器能够协助用户代理决定它应该连接到哪些来源，以及它应该获取和预处理哪些资源以提高页面性能.
