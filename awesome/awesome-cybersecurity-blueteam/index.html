    Awesome Cybersecurity Blue Team ¶

    > 为网络安全蓝队收集了一系列很棒的资源、工具和其他闪亮的东西.

    Cybersecurity blue teams 是识别信息技术系统中的安全缺陷、验证安全措施的有效性并监控系统以确保实施的防御措施在未来仍然有效的一组个人. 虽然不是排他性的，但这份清单严重偏向于 Free Software 项目和反对专有产品或公司服务. 有关攻击性 TTP，请参阅 awesome-pentest.

    衷心欢迎您的贡献和建议. （✿◕‿◕）. 请检查 Contributing Guidelines 更多细节. 这项工作已获得许可 Creative Commons Attribution 4.0 International License.

    许多网络安全专业人员通过向地方、州和联邦警务机构提供服务或以其他方式与这样做的类似机构合作，有意或无意地助长了种族主义国家暴力. 这种邪恶最常通过就业的强制机制在缺乏食物、住所或医疗保健的威胁下发生. 尽管此列表是公开的，但维护者的意图是并希望此列表支持致力于对抗如此庞大但平庸的邪恶的人和组织.

    DEFUND THE POLICE.

    Automation¶

    • Ansible Lockdown - 以信息安全为主题的 Ansible 角色的精选集合，这些角色都经过审查并积极维护.
    • Clevis - 用于自动解密的可插拔框架，经常用作 Tang 客户端.
    • DShell - 用 Python 编写的可扩展网络取证分析框架，可以快速开发插件以支持网络数据包捕获的剖析.
    • Dev-Sec.io - 服务器强化框架，提供各种基线安全配置的 Ansible、Chef 和 Puppet 实现.
    • peepdf - 可编写脚本的 PDF 文件分析器.
    • PyREBox - 基于 QEMU 的 Python 脚本化逆向工程沙箱.
    • Watchtower - 基于容器的解决方案，用于自动更新 Docker 容器基础镜像，提供无人值守的升级体验.

    Code libraries and bindings¶

    • MultiScanner - 用 Python 编写的文件分析框架，通过自动针对一组文件运行一套工具并聚合输出来帮助评估一组文件.
    • Posh-VirusTotal - VirusTotal.com API 的 PowerShell 接口.
    • censys-python - Censys REST API 的 Python 包装器.
    • libcrafter - 高级 C++ 网络数据包嗅探和制作库.
    • python-dshield - Internet Storm Center/DShield API 的 Pythonic 接口.
    • python-sandboxapi - 最小、一致的 Python API，用于构建与恶意软件沙箱的集成.
    • python-stix2 - 用于序列化和反序列化结构化威胁信息表达 (STIX) JSON 内容的 Python API，以及用于常见任务的更高级别的 API.

    Security Orchestration, Automation, and Response (SOAR)¶

    也可以看看 Security Information and Event Management (SIEM)， 和 IR management consoles.

    • Shuffle - 适用于 IT 专业人员和蓝队的图形化通用工作流（自动化）构建器.

    Cloud platform security¶

    也可以看看 asecure.cloud/tools.

    • Aaia - 在 Neo4j 的帮助下，帮助以图形格式可视化 AWS IAM 和组织.
    • Checkov - Terraform（基础设施即代码）的静态分析，以帮助检测 CIS 策略违规并防止云安全配置错误.
    • Falco - 行为活动监视器，旨在通过审核 Linux 内核并通过 Kubernetes 指标等运行时数据进行丰富，从而检测容器化应用程序、主机和网络数据包流中的异常活动.
    • Kata Containers - 使用轻量级虚拟机保护容器运行时，这些虚拟机的感觉和性能类似于容器，但使用硬件虚拟化技术作为第二层防御提供更强的工作负载隔离.
    • Principal Mapper (PMapper) - 通过能够识别 AWS 账户或 AWS 组织的 AWS Identity and Access Management (IAM) 配置中的风险的脚本和库，快速评估 AWS 中的 IAM 权限.
    • Prowler - 基于 AWS-CLI 命令的工具，用于 Amazon Web Services 帐户安全评估和强化.
    • Scout Suite - 开源多云安全审计工具，可对云环境进行安全态势评估.
    • gVisor - 应用程序内核，用 Go 编写，实现了 Linux 系统表面的很大一部分，以在应用程序和主机内核之间提供隔离边界.

    Distributed monitoring¶

    也可以看看 § Service and performance monitoring.

    • Cortex - 为 Prometheus 提供水平可扩展、高可用、多租户、长期存储.
    • Jaeger - 分布式跟踪平台后端，用于监控和排除基于微服务的分布式系统.
    • OpenTelemetry - 云原生软件的可观察性框架，包括一组工具、API 和 SDK，用于将应用程序性能指标导出到跟踪后端（以前由 OpenTracing 和 OpenCensus 项目维护）.
    • Prometheus - 最初在 SoundCloud 构建的开源系统监控和警报工具包.
    • Zipkin - 分布式跟踪系统后端，有助于收集对服务架构中的延迟问题进行故障排除所需的计时数据.

    Kubernetes¶

    也可以看看 Kubernetes-Security.info.

    • KubeSec - Kubernetes 清单的静态分析器，可以在本地运行，作为 Kuberenetes 准入控制器，或作为它自己的云服务.
    • Kyverno - 为 Kubernetes 设计的策略引擎.
    • Linkerd - 特定于 Kubernetes 的超轻型服务网格，可为 Kubernetes 应用程序增加可观察性、可靠性和安全性，而无需对应用程序本身进行任何修改.
    • Managed Kubernetes Inspection Tool (MKIT) - 查询和验证托管 Kubernetes 集群对象和集群内运行的工作负载/资源的几个常见的与安全相关的配置设置.
    • Polaris - 通过针对代码提交、Kubernetes 准入请求或已在集群中运行的实时资源运行测试来验证 Kubernetes 最佳实践.
    • certificate-expiry-monitor - 将 TLS 证书的到期时间公开为 Prometheus 指标的实用程序.
    • k-rail - Kubernetes 的工作负载策略实施工具.
    • kube-forensics - 允许集群管理员转储正在运行的 pod 及其所有容器的当前状态，以便安全专业人员可以执行离线取证分析.
    • kube-hunter - 开源工具，可以从集群外部（“攻击者的观点”）或集群内部针对 Kubernetes 集群中的安全问题运行一组测试（“猎人”）.
    • kubernetes-event-exporter - Allows exporting the often missed Kubernetes events to various outputs so that they can be used for observability or alerting purposes.

    Service meshes¶

    也可以看看 ServiceMesh.es.

    • Consul - 跨动态分布式基础架构连接和配置应用程序的解决方案，并通过 Consul Connect，通过自动 TLS 加密和基于身份的授权实现安全的服务到服务通信.
    • Istio - 开放平台，用于提供集成微服务、管理跨微服务的流量、执行策略和聚合遥测数据的统一方法.

    Communications security (COMSEC)¶

    也可以看看 Transport-layer defenses.

    • GPG Sync - 在组织或团队的所有成员之间集中和自动化 OpenPGP 公钥分发、撤销和更新.
    • Geneva (Genetic Evasion) - 新颖的实验性遗传算法，可针对民族国家级审查者发展基于数据包操纵的审查规避策略，以增加原本被阻止的内容的可用性.
    • GlobaLeaks - 免费的开源软件使任何人都可以轻松设置和维护安全的举报平台.
    • SecureDrop - 媒体组织和非政府组织可以安装开源举报人提交系统，以安全地接受来自匿名来源的文件.
    • Teleport - 允许工程师和安全专家在所有环境中统一对 SSH 服务器、Kubernetes 集群、Web 应用程序和数据库的访问.

    DevSecOps¶

    也可以看看 awesome-devsecops.

    • Bane - 用于 Docker 容器的自定义和更好的 AppArmor 配置文件生成器.
    • BlackBox - 通过使用 GnuPG 加密“静态”秘密，将秘密安全地存储在 Git/Mercurial/Subversion 中.
    • Cilium - 开源软件，用于透明地保护使用 Linux 容器管理平台（如 Docker 和 Kubernetes）部署的应用程序服务之间的网络连接.
    • Clair - 用于探测通过应用程序容器（例如 Docker）镜像引入的漏洞的静态分析工具.
    • CodeQL - 通过像数据一样对代码执行查询来发现代码库中的漏洞.
    • DefectDojo - 为 DevOps 和持续安全集成构建的应用程序漏洞管理工具.
    • Gauntlt - 在常规持续集成构建管道期间对应用程序进行渗透测试.
    • Git Secrets - 防止您将密码和其他敏感信息提交到 git 存储库.
    • SOPS - 支持 YAML、JSON、ENV、INI 和二进制格式并使用 AWS KMS、GCP KMS、Azure Key Vault 和 PGP 加密的加密文件编辑器.
    • Snyk - 查找并修复开源依赖项和容器映像中的漏洞和许可证违规.
    • SonarQube - 持续检查工具，可在自动测试期间提供详细报告，并对新引入的安全漏洞发出警报.
    • Trivy - 简单而全面的容器和其他工件的漏洞扫描器，适用于持续集成管道.
    • Vault - 用于通过统一界面安全访问机密信息（例如 API 密钥、密码或证书）的工具.
    • git-crypt - git中的透明文件加密； 您选择保护的文件在提交时加密，签出时解密.
    • helm-secrets - Helm 插件，帮助管理 Git 工作流程中的机密并将它们存储在任何地方，由 SOPS 支持.

    Application or Binary Hardening¶

    • DynInst - 用于二进制检测、分析和修改的工具，对二进制修补很有用.
    • DynamoRIO - 运行时代码操作系统，支持程序任何部分的代码转换，同时它执行，作为进程级虚拟机实现.
    • Egalito - 二进制重新编译器和检测框架，可以完全反汇编、转换和重新生成专为二进制加固和安全研究设计的普通 Linux 二进制文件.
    • Valgrind - Instrumentation framework for building dynamic analysis tools.

    Compliance testing and reporting¶

    • Chef InSpec - 用于描述安全性和合规性规则的语言，这些规则成为可以针对 IT 基础架构运行的自动化测试，以发现和报告不合规性.
    • OpenSCAP Base - 用于根据 SCAP 基线配置文件评估系统的库和命令行工具 (oscap)，以报告扫描系统的安全状况.

    Fuzzing¶

    也可以看看 Awesome-Fuzzing.

    • Atheris - 基于 libFuzzer 的覆盖引导 Python 模糊测试引擎，支持 Python 代码模糊测试以及为 CPython 编写的本机扩展.
    • FuzzBench - 免费服务，可在 Google 规模的各种真实世界基准测试中评估模糊器.
    • OneFuzz - 自托管的模糊测试即服务 (FaaS) 平台.

    Policy enforcement¶

    • AllStar - GitHub 应用程序安装在组织或存储库上以设置和实施安全策略.
    • Conftest - 帮助您针对结构化配置数据编写测试的实用程序.
