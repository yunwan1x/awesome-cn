    Implementations¶

    JavaScript/TypeScript¶

    • graphql-js - 用于 JavaScript 的 GraphQL 参考实现.
    • graphql-jit - 使用 JIT 编译器执行 GraphQL.

    Clients¶

    • apollo-client - 适用于每个 UI 框架和 GraphQL 服务器的功能齐全、生产就绪的缓存 GraphQL 客户端.
    • graphql-request - 适用于 Node 和浏览器的最小 GraphQL 客户端.
    • typescript-graphql-request - 使用 GraphQL 请求作为全类型 SDK.
    • graphql-zeus - GraphQL Zeus 为“JavaScript”或“TypeScript”创建自动完成客户端库，为强类型查询提供自动完成功能.
    • graphqurl - curl for GraphQL with autocomplete, subscriptions and GraphiQL. Also a dead-simple universal javascript GraphQL client.
    • aws-amplify - 由 Amazon 开发的用于缓存、分析等的客户端库，包括获取 GraphQL 查询的方法.
    Frontend Framework Integrations¶
    • vue-apollo - VueJS 的 Apollo/GraphQL 集成.
    • apollo-angular - 用于 Angular 和每个 GraphQL 服务器的功能齐全、生产就绪的缓存 GraphQL 客户端.
    • svelte-apollo - Apollo GraphQL 的 Svelte 集成.
    • ember-apollo-client - 用于 Apollo 客户端和 GraphQL 的 ember-cli 插件.
    • apollo-elements - 适用于任何前端框架的 GraphQL Web 组件.
    React¶
    • react-apollo - 核心@apollo/client 库提供了与 React 的内置集成.
    • relay - Relay 是一个 JavaScript 框架，用于构建数据驱动的 React 应用程序.
    • urql - 一个简单的 React 缓存 GraphQL 客户端.
    • graphql-hooks - 具有缓存和服务器端渲染支持的最小钩子优先 GraphQL 客户端.
    • gqless - 一个没有查询的 GraphQL 客户端✨
    • mst-gql - mobx-state-tree 和 GraphQL 的绑定.
    • micro-graphql-react - 用于将 GraphQL 添加到 React 的轻量级实用程序. 成分. 包括简单的缓存并使用 GET 请求，这些请求可以通过 service-worker 额外缓存.

    Servers¶

    • apollo-server - 符合规范且生产就绪的 JavaScript GraphQL 服务器，可让您以模式优先的方式进行开发. 专为 Express、Connect、Hapi、Koa 等而构建.
    • express-graphql - GraphQL Express 中间件.
    • hapi-graphql - 使用 Hapi 创建一个 GraphQL HTTP 服务器.
    • hapi-plugin-graphiql - 用于 GraphiQL 集成的 HAPI 插件.
    • graphql-api-koa - 从头开始​​实现 GraphQL.js 并支持原生 ESM 的 GraphQL Koa 中间件.
    • koa-graphql --GraphQL Koa 中间件.
    • gql - 适用于 Deno 的通用 GraphQL HTTP 中间件.
    • mercurius - Fastify 的 GraphQL 插件.
    • graphql-yoga - 功能齐全的 GraphQL 服务器，专注于轻松设置、性能和出色的开发人员体验.
    • graphitejs - GraphQL 框架 NodeJS.
    • graphql-helix - 高度进化的 GraphQL HTTP 服务器.
    Databases & ORMs¶
    • graphql-sequelize - GraphQL 的 Sequelize 助手.
    • graphql-bookshelf - 围绕 BookshelfJS 模型定义 GraphQL 模式的一些帮助.
    • join-monster - 用于批量数据获取的 GraphQL-to-SQL 查询执行层.
    PubSub¶
    • graphql-ably-pubsub - GraphQL 能够发布变更更新并通过订阅查询订阅结果的 Ably PubSub 实现.

    Custom Scalars¶

    • graphql-scalars - 自定义 GraphQL 标量库，用于创建精确的类型安全 GraphQL 模式.
    • graphql-tools-types - Custom GraphQL types for use with GraphQL-Tools (Void, Int, Float, String, Date, UUID, JSON).

    Type¶

    • type-graphql - 使用类和装饰器，使用 TypeScript 创建 GraphQL 模式和解析器！
    • graphql-nexus - 代码优先、类型安全、GraphQL 模式构建.
    • graphql-code-generator：GraphQL 代码生成器，灵活支持自定义插件和模板，如 TypeScript（前端和后端）、React Hooks、解析器签名等.
    • giraphql - 基于插件的架构构建器，用于在 TypeScript 中创建代码优先的 GraphQL 架构.

    Miscellaneous¶

    • graphql-tools - 用于构建和维护 GraphQL-JS 服务器的工具库.
    • graphql-tag - 解析 GraphQL 查询的 JavaScript 模板文字标签.
    • graphql-compose - 允许您通过插件从不同数据源构建灵活的 graphql 模式的工具.
    • graphql-modules - 按模块或功能将 GraphQL 服务器分成更小的、可重用的部分.
    • graphql-shield - 一个帮助为 graphql api 创建权限层的库.
    • graphql-let - 直接从 GraphQL 文档导入受类型保护的代码生成结果的 webpack 加载器
    • graphql-config - 所有 GraphQL 工具的一种配置（大多数工具、编辑器和 IDE 都支持）.
    • graphql-cli - 用于常见 GraphQL 开发工作流的命令行工具.
    • graphql-toolkit - 一组用于更快开发 GraphQL 工具（架构和文档加载、架构合并等）的实用程序.
    • graphql-mesh - 使用 GraphQL 查询语言访问不运行 GraphQL 的远程 API（以及运行 GraphQL 的 API）中的数据.
    • sofa - 从您的 GraphQL API 生成 REST API.
    • graphback - 使用数据模型将 GraphQLCRUD API 层添加到 GraphQL 服务器的框架和 CLI.
    • graphql-middleware - 在中间件函数中拆分 GraphQL 解析器.
    • graphql-relay-js - 一个帮助构建支持 react-relay 的 graphql-js 服务器的库.
    • graphql-normalizr - 规范化 GraphQL 响应以保留在客户端缓存/状态中.
    • babel-plugin-graphql - 编译 GraphQL 标记模板字符串的 Babel 插件.
    • eslint-plugin-graphql - 一个 ESLint 插件，用于根据架构检查您的 GraphQL 字符串.
    • graphql-ws - 一致、零依赖、懒惰、简单、基于 WebSocket 协议的 GraphQL 兼容服务器和客户端.
    • graphql-live-query - 使用 JavaScript 进行实时 GraphQL 实时查询.
