    Awesome ebpf

    Awesome eBPF ¶

    > 与 eBPF 相关的精彩项目的精选列表.

    BPF，就像_Berkeley Packet Filter_一样，是一个内核虚拟机，运行从用户空间传递过来的程序. 最初在 BSD 上实现，然后在 Linux 上实现，（现在是传统的）“经典 BPF”或 cBPF 机器将与 tcpdump 等工具一起使用，用于过滤内核中的数据包以避免无用的副本到用户空间. 最近，Linux 中的 BPF 基础设施已经完全重新设计并赋予“扩展 BPF”或 eBPF 生命，它获得了新功能（安全和终止检查、程序的 JIT 编译、持久映射、标准库、硬件卸载支持等），现在用于许多任务. 在非常低的级别 (XDP) 处理数据包、跟踪和监视系统上的事件或对 cgroup 实施访问控制只是 eBPF 带来性能、可编程性和灵活性的几个例子.

    Recently Cilium 推出了一个关于 eBPF 的很棒的网站，名为 ebpf.io . 它的用途与此列表类似，具有 an introduction to eBPF 和链接到 related projects.

    > 注意：eBPF 是一项令人兴奋的技术，其生态系统也在不断发展. 我们希望 you 提供帮助，使这个很棒的列表保持最新状态，并尽我们所能提高其信噪比. 请放心离开 any feedback.

    • bcc/tools - 这些工具本身可以被视为 BPF 程序的示例用例，主要用于跟踪和监控. 某些 Linux 发行版已经打包了 bcc 工具.
    • MPLSinIP sample - 一个被大量评论的示例，演示了如何在 IP 中封装和解封装 MPLS. 代码是为那些刚接触 BPF 开发的人注释的.
    • ebpf-samples - 从多个项目中收集的已编译（作为 ELF 目标文件）样本的集合，主要用作用户空间验证器的测试用例.
    • ebpf-kill-example - 一个完整记录和测试的 eBPF 探针示例，它记录所有强制终止并将它们打印在用户空间中.

    eBPF Workflow: Tools and Utilities¶

    bcc¶

    • bcc - 框架和工具集 - 一种处理 BPF 程序的方法，特别是用于跟踪和监视. 还包括一些可以帮助检查系统上的地图或程序的实用程序.
    • P4 compiler for BPF targets for bcc - 受限 C 的替代方案.
    • Lua front-end for BCC - 另一个 C 的替代品，甚至是 bcc 中使用的大多数 Python 代码.

    iproute2¶

    • iproute2 - 包含 Linux 网络管理工具的软件包. 特别是它包含了tc，用于管理eBPF过滤器和动作，以及ip，用于管理XDP程序. 大多数与 BPF 相关的代码都在 lib/bpf.c 中.
    • iproute2-next - 开发树，与 net-next 同步.

    LLVM¶

    • LLVM - 包含在 eBPF 工作流中使用的几个工具. 可以从以下位置检索 Ubuntu/Debian 最新版本的快照 here.

    • clang 用于将 C 编译为 ELF 格式（clang v3.7.1+）下的 eBPF 目标文件. BPF 后端添加了 this commit.

    • llvm-objdump 用于以人类可读的格式转储目标文件的内容，可能带有初始 C 源代码 (llvm-objdump v4.0+).
    • llvm-mc 用于将 LLVM 中间表示编译为 eBPF 目标文件，以便可以从 C 编译为 eBPF 程序集，修改程序集，然后编译为 ELF 文件.

    libbpf¶

    • libbpf - 用于处理 BPF 对象（程序和映射）以及操作包含它们的 ELF 对象文件的 AC 库. 它随内核一起提供，并且 mirrored on GitHub.
    • libbpf-bootstrap - 使用 libbpf 和 BPF CO-RE 进行 BPF 应用程序开发的脚手架.

    bpftool and Other Tools from the Kernel Tree¶

    • bpftool - 还有一些内核树中的其他工具，在 linux/tools/net/ 对于 4.15 之前的版本，或 linux/tools/bpf/ 之后：

    • bpftool - 通用实用程序，可用于从用户空间与 eBPF 程序和映射进行交互，例如显示、转储、加载、反汇编、固定程序，或显示、创建、固定、更新、删除地图，或附加和分离程序到 cgroup.

    • bpf_asm - 一个最小的 cBPF 汇编器.
    • bpf_dbg - cBPF 程序的小型调试器.
    • bpf_jit_disasm - 用于两种 BPF 风格的反汇编器，对于 JIT 调试非常有用.

    User Space eBPF¶

    • uBPF - 用 C 编写.包含一个解释器、一个用于 x86_64 架构的 JIT 编译器、一个汇编器和一个反汇编器.
    • A generic implementation - 支持 FreeBSD 内核、FreeBSD 用户空间、Linux 内核、Linux 用户空间和 macOS 用户空间. 用于 VALE software switch's BPF extension module.
    • rbpf - 用 Rust 编写. 适用于 Linux、macOS 和 Windows 的解释器，以及适用于 Linux 下 x86_64 的 JIT 编译器.
    • PREVAIL - A user space verifier for eBPF using an abstract interpretation layer，支持循环.
    • oster - 用 Go 编写. 通过将 eBPF 附加到 uprobes 来跟踪 Go 程序执行的工具.

    Testing in Virtual Environments¶

    The Code¶

    • linux/include/linux/bpf.h - 和 linux/include/uapi/bpf.h: 与 eBPF 相关的定义，分别用于内核和与用户空间程序的接口.
    • linux/include/linux/filter.h - 和 linux/include/uapi/filter.h：用于运行 BPF 程序本身的信息.

    • linux/kernel/bpf/ - 该目录包含大部分与 BPF 相关的代码. 特别是，这些文件值得关注：

    • syscall.c - 系统调用允许的不同操作，例如程序加载或地图管理.

    • core.c - BPF 解释器.

    • verifier.c - BPF 验证器.

    • linux/net/core/filter.c - 与网络相关的功能和 eBPF 助手（TC、XDP 等）； 还包含将 cBPF 字节码迁移到 eBPF 的代码（所有 cBPF 程序在最近的内核中都被转换为 eBPF）.

    • linux/kernel/trace/bpf_trace.c - 与跟踪和监控相关的函数和 eBPF 助手（kprobes、tracepoints 等）.
    • The JIT compilers are under the directory of their respective architectures, such as file linux/arch/x86/net/bpf_jit_comp.c 对于 x86. 用于硬件卸载的 JIT 编译器例外，它们位于驱动程序中，例如 linux/drivers/net/ethernet/netronome/nfp/bpf/jit.c 为 Netronome NFP.
    • linux/net/sched/ - 特别是在文件 act_bpf.c（动作）和 cls_bpf.c（过滤器）中：与 BPF 动作和带有 TC 的过滤器相关的代码.
    • linux/kernel/seccomp.c
    • linux/net/core/dev.c - 包含函数dev_change_xdp_fd()，在从用户空间加载到内核后，通过 Netlink 命令调用该函数以将 XDP 程序挂接到设备. 该函数依次使用来自相关驱动程序的回调.

    • @IOVisor Twitter account
    • The XDP Collaboration Project - 一个 GitHub 存储库，其中包含有关 XDP 未来发展的注释和想法.
